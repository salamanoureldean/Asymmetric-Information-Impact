# CELL-DEVS Information Impact

## Introduction
This project implements a Cell-DEVS simulation of information impact in a social network using Cadmium.  
The model simulates how influence spreads across a grid under different initial configurations and neighborhood conditions.

The repository includes:
- the Cell-DEVS model source code
- scenario configuration files
- build and execution scripts
- generated log files for each experiment
- recorded simulation videos for selected scenarios

## Dependencies
- This project assumes that Cadmium is installed and accessible through the environment variable $CADMIUM.

You will also need:

- cmake
- make
- a C++17-compatible compiler
- a Linux or Linux-like shell environment

## Build Instructions
- From the project root, build the simulation with: ./build_sim.sh or bash build_sim.sh depending on your environment.

## Run predefined scenarios directly
- Execute the helper scripts for each experiment directly, it will build and run each scenario:
- ./run_base_scenario.sh or bash run_base_scenario.sh
- ./run_clustered_scenario.sh or bash run_clustered_scenario.sh
- ./run_edge_wrap_scenario.sh or bash run_edge_wrap_scenario.sh
- ./run_single_seed_scenario.sh or bash run_single_seed_scenario.sh

## Configuration Files

- The simulation scenarios are controlled through the JSON files in the config/ directory.

Available configurations:

- social_config_base_scenario.json
- social_config_clustered.json
- social_config_edge_wrap.json
- social_config_single_seed.json

The visualization-related settings are defined in:

- socialVisualization_config.json

## Web Viewer Use
- Combine any of the .csv files with the socialVisualization_config.json to visualize the simulation of different scenarios in the web viewer.

## Output Files

- Simulation results are written to the log/ directory as CSV files.

Generated outputs include:

- social_log_base_scenario_log.csv
- social_log_clustered_log.csv
- social_log_edge_wrap_log.csv
- social_log_single_seed_log.csv

These files contain the recorded state evolution of the simulation and can be used for analysis or visualization.

- Recorded Simulations

This repository also includes recorded simulation videos:

- base_scenario_simulation.webm
- clustered_scenario_simulation.webm
- edge_wrap_scenario_simulation.webm
- single_seed_scenario_simulation.webm

## Notes
- The build/ and bin/ folders are generated during compilation.
- The shell scripts provided in the repository are the recommended way to run the predefined experiments.
- If cmake or make is not found, install them before attempting to build.
- If $CADMIUM is not set, Cadmium is not configured correctly in the environment.

## Asymmetric Cell-DEVS extension

This repository now includes a first asymmetric Cell-DEVS implementation path for the term project.

New source files:
- include/socialAsymmCell.hpp
- include/socialAsymmCoupled.hpp
- main/main_asymm.cpp

New asymmetric scenario files:
- config/social_asymm_two_communities.json
- config/social_asymm_bridge.json
- config/social_asymm_influencers.json

New run scripts:
- ./run_two_communities_asymm.sh
- ./run_bridge_asymm.sh
- ./run_influencers_asymm.sh

Notes:
- The asymmetric version follows the Cadmium / asymmetric Cell-DEVS workflow from the course paper: cells are defined in C++ and per-cell weighted neighborhoods are defined in JSON.
- The new rule uses weighted exposure from neighbors in state 1 or 2.
- By default the implementation normalizes active incoming weight by total incoming weight before applying thresholds.
- The asymmetric executable is built as bin/social_network_impact_asymm.
