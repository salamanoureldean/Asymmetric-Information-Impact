# Asymmetric Cell DEVS - Social Network Impact Simulation

## Introduction
This project implements an Asymmetric Cell-DEVS simulation of social network dynamics using Cadmium.  
The model simulates how influence and information spreads across a social network under different initial conditions and community structures.

The repository includes:
- the Asymmetric Cell-DEVS model source code
- scenario configuration files
- build and execution scripts
- Python utilities for log processing
- generated log files for each experiment
- recorded simulation videos

## Dependencies
- This project assumes that Cadmium is installed and accessible through the environment variable $CADMIUM.

You will also need:

- cmake
- make
- a C++17-compatible compiler
- a Linux or Linux-like shell environment

## Build Instructions
- From the project root, build the simulation with: ./build_sim.sh or bash build_sim.sh depending on your environment.

## Run Predefined Scenarios
PREFERRED:
In WSL, execute the helper scripts for each scenario to build and run the simulation:

- `./run_base_scenario.sh` - Base scenario with standard configuration
- `./run_boundary_seed_scenario.sh` - Boundary seed scenario
- `./run_strong_cross_community_scenario.sh` - Strong cross-community scenario
- `./run_weak_bridges_scenario.sh` - Weak bridges scenario

Or use `bash` if on a system where direct execution is not available:
```bash
bash run_base_scenario.sh
bash run_boundary_seed_scenario.sh
bash run_strong_cross_community_scenario.sh
bash run_weak_bridges_scenario.sh
```

## Configuration Files

Simulation scenarios are controlled through JSON configuration files in the `config/` directory.

Available configurations:

- `social_config_base_scenario.json` - Base scenario configuration
- `social_config_boundary_seed.json` - Boundary seed scenario configuration
- `social_config_strong_cross_community.json` - Strong cross-community scenario configuration
- `social_config_weak_bridges.json` - Weak bridges scenario configuration

Visualization settings are defined in:

- `socialVisualization_config.json`

Configuration JSON generators (Python scripts) for each scenario are available in the `generators/`

## Web Viewer Visualization

Combine any generated viewer log file with `socialVisualization_config.json` to visualize simulation scenarios in the web viewer.

## Output Files

Simulation results are written to the `log/` directory as CSV files:

- `social_log_base_scenario_viewer_log.csv`
- `social_log_boundary_seed_viewer_log.csv`
- `social_log_strong_cross_community_viewer_log.csv`
- `social_log_weak_bridges_viewer_log.csv`

These files contain the recorded state evolution of the simulation and can be used for analysis or visualization.

## Recorded Simulations

The `Simulation_videos/` directory contains recorded simulation videos for the different scenarios.

## Notes
- The build/ and bin/ folders are generated during compilation.
- The shell scripts provided in the repository are the way to run the predefined experiments.
- If cmake or make is not found, install them before attempting to build.
- If $CADMIUM is not set, Cadmium is not configured correctly in the environment.

## Asymmetric Cell-DEVS extension

This repository now includes a first asymmetric Cell-DEVS implementation path for the term project.

New source files:
- include/socialAsymmCell.hpp
- include/socialAsymmCoupled.hpp
- main/main_asymm.cpp

New asymmetric scenario files:
- config/social_asymm_two_communities.json
- config/social_asymm_bridge.json
- config/social_asymm_influencers.json

New run scripts:
- ./run_two_communities_asymm.sh
- ./run_bridge_asymm.sh
- ./run_influencers_asymm.sh

Notes:
- The asymmetric version follows the Cadmium / asymmetric Cell-DEVS workflow from the course paper: cells are defined in C++ and per-cell weighted neighborhoods are defined in JSON.
- The new rule uses weighted exposure from neighbors in state 1 or 2.
- By default the implementation normalizes active incoming weight by total incoming weight before applying thresholds.
- The asymmetric executable is built as bin/social_network_impact_asymm.
